# 🔐 Módulo 7: Seguridad en la Nube, Móviles e IdC

## 📖 Resumen

Este módulo aborda la seguridad en entornos modernos:

* Computación en la nube
* Dispositivos móviles
* Gestión de identidad y accesos

## 🧠 Análisis

La seguridad ha evolucionado hacia entornos distribuidos.
Uno de los principales riesgos es la **mala configuración en servicios cloud**, lo que puede exponer datos sensibles.

Los dispositivos móviles aumentan la superficie de ataque debido a su uso mixto (personal y corporativo).

## 🔍 Puntos clave

* Configuraciones incorrectas = principal vulnerabilidad
* Importancia del control de accesos (IAM)
* Riesgos en aplicaciones móviles

## 🛠️ Aplicación

* Auditoría de permisos en la nube
* Evaluación de seguridad en apps móviles
* Implementación de autenticación segura

# ⚙️ Módulo 8: Técnicas de Post-Explotación

## 📖 Resumen

Se enfoca en las acciones después de comprometer un sistema.

## 🧠 Análisis

El acceso inicial no representa el mayor riesgo, sino lo que el atacante puede hacer después.

## 🔍 Técnicas principales

* Escalada de privilegios
* Movimiento lateral
* Persistencia
* Recolección de datos

## 🛠️ Aplicación

* Mantener acceso al sistema comprometido
* Evadir mecanismos de detección
* Extraer credenciales y datos sensibles

## ⚠️ Conclusión

Una pequeña vulnerabilidad puede escalar a un compromiso total del sistema.

# 📝 Módulo 9: Informes y Comunicación

## 📖 Resumen

Se centra en la documentación de vulnerabilidades.

## 🧠 Análisis

Encontrar fallos no es suficiente.
El valor real está en saber comunicarlos correctamente.

## 🔍 Elementos clave

* Claridad en la redacción
* Adaptación a la audiencia
* Priorización de riesgos

## 🛠️ Aplicación

* Creación de reportes profesionales
* Explicación del impacto de vulnerabilidades
* Recomendaciones claras

## ⚠️ Conclusión

Un mal informe reduce el impacto del trabajo técnico.

# 🧪 Módulo 10: Herramientas y Análisis de Código

## 📖 Resumen

Se enfoca en detectar vulnerabilidades directamente en el código.

## 🧠 Análisis

Prevenir vulnerabilidades en desarrollo es más eficiente que corregirlas en producción.

## 🔍 Tipos de análisis

* Análisis estático
* Análisis dinámico

## 🛠️ Aplicación

* Uso de herramientas automatizadas
* Revisión manual de código
* Identificación de vulnerabilidades comunes:

  * Inyecciones
  * Errores de validación
  * Fallos de lógica

## ⚠️ Conclusión

Las herramientas ayudan, pero el análisis humano es esencial.

# 🚀 Conclusiones Generales

Estos módulos permiten comprender el hacking ético desde una perspectiva completa:

## ✅ Aprendizajes clave

* La seguridad moderna incluye cloud y dispositivos móviles
* El impacto real ocurre en la post-explotación
* La comunicación es esencial en ciberseguridad
* El análisis de código previene vulnerabilidades

## 💡 Reflexión final

Un hacker ético no solo detecta fallos, sino que:

* Entiende su impacto
* Los explota de forma controlada
* Propone soluciones claras

## 🎯 Resultado

Formación más cercana a escenarios reales de ciberseguridad.
